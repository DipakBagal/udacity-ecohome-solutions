## â“ Why Aren't These in Git?

1. **Best Practice:** Don't commit generated/binary files to version control
2. **Size:** Database and vector store add 5-10 MB+ to repository
3. **Reproducibility:** Anyone can generate identical data by running the notebooks
4. **Security:** Databases might contain sensitive data in real deployments
5. **Flexibility:** Users can customize data generation parameters

## ğŸ”„ Regenerating Data

To regenerate the databases:

1. Delete existing files:
   ```bash
   # Windows PowerShell
   Remove-Item ecohome.db -ErrorAction SilentlyContinue
   Remove-Item -Recurse -Force chroma_db -ErrorAction SilentlyContinue
   
   # Mac/Linux
   rm -f ecohome.db
   rm -rf chroma_db/
   ```

2. Run setup notebooks again:
   ```bash
   jupyter notebook 01_db_setup.ipynb
   jupyter notebook 02_rag_setup.ipynb
   ```
